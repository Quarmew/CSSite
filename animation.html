Что такое анимации в CSS?

CSS3-анимация придаёт сайтам динамичность. В отличие от CSS3-переходов, создание анимации базируется на ключевых кадрах, которые позволяют автоматически воспроизводить и повторять эффекты на протяжении заданного времени, а также останавливать анимацию внутри цикла.

Transition

Transition позволяет определять переходное состояние между двумя состояниями элемента.

Есть состояние элемента под названием hover. Оно реализуется если навести курсором мыши на элемент. Для улучшение UX для кнопок зачастую добавляют анимацию наведения и нажатия. Для этого достаточно изменение одного двух свойств кнопки.

В hover мы прописывает измененный стиль. При наведениии всё работает резко без плавности. Для этого в обычный стиль элемента мы прописываем свойство transition:

Без transition
.block{
    height: 50px;
    width: 100px;
    background: #1a909f;
}
.block:hover{
    width: 200px;
}

Классно, работает, но не дружелюбно. Добавим transition. У него указываем сначало свойство, которое мы хотим изменить, затем длительность.

.block{
    width: 100px;
    height: 50px;
    background: #2D9A82;
    transition: width 2s;
}
.block:hover{
    width: 200px;
}

Получился медленно тянущийся элемент, но уже лучше. Теперь приведу пример разной длительности анимации.

Так же у этого свойства есть возможность задать задержку и временную функцию.

При указании задержки мы как не странно отсрочиваем начала изменений.

Временная фунция задаётся следующими значениями:

  • linear

    Равномерное изменение без колебаний

  • ease

    Переход начинается медленно, быстрый разгон, замедление.

  • ease-in

    Медленное начало, плавное ускорение.

  • ease-out

    Быстрое начало и плавное замедление в конце

  • ease-in-out

    Медленное начало и медленный конец.

  • steps()

    В скобках указываются количество ходов и также вид уже самих ходов.

  • inherit, initial, unset

    Наследует от родителя, по умолчанию, убирает значение.

  • cubic-bezier()

    Это целая функция для расчёта кривой. Ей можно задать все остальные значения этого временной функции.

Keyframes

@keyframes имя анимации {
    from {...}
    to {...}
}

Создание анимации начинается с установки ключевых кадров правила @keyframes. Кадры определяют, какие свойства на каком шаге будут анимированы. Каждый кадр может включать один или более блоков объявления из одного или более пар свойств и значений.

Из структуры выше можно понять что нужно указать название и записать значения в from и to, но сами from и to можно заменить на проценты. То есть расписанить эту анимацию на 3, 4 ,5 и сколько угодно действий исползуя проценты от 0 до 100 (где 0 - начальная позиция или первый кадр, а 100 - конечная позиция или конечный кадр).

@keyframes breath{
    0%{
        width: 115px;
        animation-timing-function: ease-out;
    }
    25%{
        width: 130px;
        animation-timing-function: linear;
    }
    50%{
        width: 125px;
        animation-timing-function: linear;
    }
    75%{
        width: 110px;
        animation-timing-function: ease-in-out;
    }
    100%{
        width: 100px;
        animation-timing-function: ease-in-out;
    }
}

Мы указали процентные значения. изменяется width и у каждого момента своя временная функция.

Теперь, чтобы воспользоваться этой анимацией необходимо у элемента, на которого будет распространяться эта анимация, установить свойство animation

Animation

Подвойства:

  • animation-name

    Название используемой анимации(у нас она названа breath).

  • animation-duration

    Длительность одного цикла анимации

  • animation-timing-function

    Ускорение анимации. Мы указали внутри анимации.

  • animation-delay

    Задержка перез началом анимации

  • animation-iteration-count

    Количество повторений. Можно ввести infinity - бесконечное повторение анимации

  • animation-direction

    Даёт возможность при каждом повторе анимации идти по альтернативному пути, либо сбросить все значения и повторить анимацию.

  • animation-fill-mode

    Настраивает значения, используемые анимацией, до и после исполнения.

  • animation-play-state

    Позволяет приостановить и возобновить анимацию.

.breath-block{
    animation: breath 3s;

Задал name, duration и animation-iteration-count.

И это при учёте того что изменяется только одно свойство элемента - width.

CSS-анимации VS JS-анимации

Самую простую анимацию можно создавать как с помощью CSS, так и с помощью JavaScript, однако объем затрат труда и времени будет разным. У каждого варианта есть свои достоинства и недостатки, но есть хорошие правила, которым нужно следовать:

  • Используйте CSS при работе с самодостаточными состояниями элементов пользовательского интерфейса небольшого размера.

    Переходы и анимация, выполненные средствами CSS, идеально подходят для вывода меню навигации на экран сбоку или отображения подсказки. JavaScript можно использовать для контроля состояния, однако сама анимация будет выполнена средствами CSS.

  • Используйте JavaScript, когда требуется высокая степень контроля над анимацией.

    Когда нужно реализовать динамическое отслеживание положения касания или анимацию, которую необходимо будет остановить, приостановить, замедлить или запустить в обратном направлении, следует пользоваться JavaScript.
